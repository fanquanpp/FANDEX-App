二、内容源与修改约束

	FANDEX 仓库（https://github.com/fanquanpp/FANDEX.git ）为全部项目的根本内容源，所有文档、索引及数据均以此为唯一蓝本。
	该仓库仅允许只读引用，禁止直接修改或覆盖。其余仓库如需变更内容，须从 FANDEX 仓库复制后在其独立仓库内进行适配与优化。
	三个仓库代码互相独立，内容互相关联，内容基准统一以 FANDEX 仓库为准。

三、各仓库职责与功能定位

	FANDEX         基于 GitHub Pages 构建的云端在线查阅平台，为最早建立的内容仓库，文档体系最为完善
	FANDEX-App     Android 平台纯代码语法速查工具，无网络依赖，聚焦编程语言语法格式查阅，不包含教学讲解内容
	FANDEX-Web     Web App 版本

四、FANDEX-App 内容规则

	1. 定位：纯粹的代码语法和语法格式查看阅览工具
	2. 仅收录编程语言/查询语言/标记语言的语法格式文档
	3. 不收录以下类型文档：
	   - 教学讲解（概述、概念阐述、发展历史）
	   - 环境配置（安装指南、环境搭建、工具配置）
	   - 项目实战（项目示例、实战案例、综合应用）
	   - 理论知识（算法理论、数学推导、设计原理）
	   - 最佳实践（性能优化、调试技巧、工程规范）
	   - Vue 组件型文件（无法在 App 原生渲染器中显示）
	4. 当前收录模块（17 个）：
	   编程语言：C、C++、C#、Go、Java、JavaScript、Kotlin、Lua、Python、TypeScript
	   前端技术：CSS
	   数据库：MySQL、PostgreSQL、Redis、SQL
	   工具链：Git
	   标记语言：Markdown
	5. 文档格式遵循 FANDEXAPP文档格式模板（不可删除）.md 中定义的语法速查模板

五、视觉风格与功能特性

	三个仓库风格各有特色，视觉语言保持关联性，均具备深色与浅色双模式。
	FANDEX 具备双端适配页面、移动端专属目录功能及自动构建部署能力。
	FANDEX-App 具备完全离线查阅能力，聚焦纯粹的代码语法速查体验，支持中英日三语界面切换。
	FANDEX-Web

六、项目更新日志规则

	FANDEX 更新日志规则：
		1. 新模块、新功能、新页面的增加以及重构（前端布局、前端排版）属于大版本更新，版本号由 1.x.x 升至 2.x.x，须详细说明更新内容
		2. 小 BUG 修复（文档纠错、功能按钮位置调整、颜色优化不改变 UI 风格、按钮颜色图形优化）属于小更新，版本号由 1.0.x 升至 1.1.x，更新日志简要书写
		3. 同一问题或其所属范围内的多次修复，版本号由 1.x.0 升至 1.x.1，更新日志写：修复了一些 BUG
		4. 仅第 1 条独立作为更新版本书写；第 2 条写在第 1 条内；第 3 条写在第 2 条内，呈包含关系
		5. 前端页面底部信息区域须标注最新版本号与更新日期，不包含更新内容

	FANDEX-App 更新日志规则：
		1. 新模块、新功能、新页面的增加以及重构（前端布局、前端排版）属于大版本更新，版本号由 1.x.x 升至 2.x.x，须详细说明更新内容
		2. 小 BUG 修复（文档纠错、功能按钮位置调整、颜色优化不改变 UI 风格、按钮颜色图形优化）属于小更新，版本号由 1.0.x 升至 1.1.x，更新日志简要书写
		3. 同一问题或其所属范围内的多次修复，版本号由 1.x.0 升至 1.x.1，更新日志写：修复了一些 BUG
		4. 版本留存规则：仅留存每个大版本（1.0.0、1.1.0、2.0.0、3.0.0 等）的最终安装包（如 1.0.3、2.2.4、3.0.8）；大版本初始安装包（1.0.0、2.0.0、3.0.0 等）须单独留存，与最终安装包分开存放
		5. 大版本初始版本的更新日志单独存在，该大版本后续更新写在另一条更新日志下

	FANDEX-Web 更新日志规则：
		1. 仓库记录每一次更新日志，须附带详细日期与内容，无论版本大小
