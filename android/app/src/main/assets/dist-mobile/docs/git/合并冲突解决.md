# 合并冲突解决

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 冲突标记格式

**冲突标记结构**：Git 在冲突文件中插入的标记
`<<<<<<< HEAD ... ======= ... >>>>>>> <分支名>`

```text
# 冲突标记格式
<<<<<<< HEAD
当前分支的内容
=======
合并分支的内容
>>>>>>> feature
```

---

## 冲突解决标准流程

**尝试合并**：触发合并操作
`git merge <分支名>`

```bash
# 合并 feature 分支到当前分支
git merge feature;
```

**查看冲突文件**：列出所有冲突文件
`git status`

```bash
# 查看冲突状态
git status;
# Unmerged paths:
#   both modified:   src/index.js
```

**标记冲突已解决**：添加解决后的文件
`git add <file>`

```bash
# 将解决冲突后的文件加入暂存区
git add src/index.js;
```

**完成合并提交**：提交合并结果
`git commit`

```bash
# 提交合并结果
git commit;
```

---

## 查看冲突详情

**列出冲突文件**：仅显示有冲突的文件名
`git diff --name-only --diff-filter=U`

```bash
# 列出所有冲突文件
git diff --name-only --diff-filter=U;
```

**查看冲突内容**：显示冲突的具体差异
`git diff`

```bash
# 查看冲突内容
git diff;
```

**使用合并工具**：启动可视化合并工具
`git mergetool`

```bash
# 启动配置的合并工具
git mergetool;
```

---

## 选择一方版本

**采用当前分支版本**：冲突时使用 ours
`git checkout --ours <file>`

```bash
# 采用当前分支版本的 src/config.js
git checkout --ours src/config.js;
```

**采用合并分支版本**：冲突时使用 theirs
`git checkout --theirs <file>`

```bash
# 采用合并分支版本的 src/styles.css
git checkout --theirs src/styles.css;
```

---

## 合并策略选项

**合并双方修改**：冲突时合并双方内容
`git merge -X union <分支名>`

```bash
# 使用 union 策略合并双方修改
git merge -X union feature;
```

**冲突时采用当前分支**：所有冲突使用 ours
`git merge -X ours <分支名>`

```bash
# 冲突时总是采用当前分支
git merge -X ours feature;
```

**冲突时采用合并分支**：所有冲突使用 theirs
`git merge -X theirs <分支名>`

```bash
# 冲突时总是采用合并分支
git merge -X theirs feature;
```

---

## 放弃合并

**放弃当前合并**：回到合并前状态
`git merge --abort`

```bash
# 放弃当前合并操作
git merge --abort;
```

**硬重置放弃合并**：丢弃所有合并相关修改
`git reset --hard HEAD`

```bash
# 强制回到合并前的 HEAD 状态
git reset --hard HEAD;
```

## 多文件冲突处理

**批量采用 ours**：所有冲突文件使用当前分支版本
`git checkout --ours .`

```bash
# 批量采用当前分支版本
git checkout --ours .;
```

**批量采用 theirs**：所有冲突文件使用合并分支版本
`git checkout --theirs .`

```bash
# 批量采用合并分支版本
git checkout --theirs .;
```

**逐文件处理冲突**：遍历冲突文件
`for file in $(git diff --name-only --diff-filter=U)`

```bash
# 遍历所有冲突文件逐个处理
for file in $(git diff --name-only --diff-filter=U); do
    echo "Conflict in: $file"
    # 手动处理每个文件
done
```

## 重命名冲突

**查看重命名情况**：检测重命名冲突
`git diff --name-status --diff-filter=R`

```bash
# 查看重命名的文件
git diff --name-status --diff-filter=R;
```

## 子模块冲突

**查看子模块指向的提交**：检查子模块冲突
`git ls-tree HEAD <子模块路径>`

```bash
# 查看子模块指向的提交
git ls-tree HEAD path/to/submodule;
```

**解决子模块冲突**：选择正确的子模块提交
`git checkout <提交哈希>`

```bash
# 1. 进入子模块目录
cd path/to/submodule;
# 2. 切换到正确的提交
git checkout correct-commit;
# 3. 返回主仓库
cd ..;
# 4. 添加子模块
git add path/to/submodule;
```

---

## 预合并检查

**测试合并（不提交）**：检查是否会产生冲突
`git merge --no-commit --no-ff <分支名>`

```bash
# 测试合并但不提交
git merge --no-commit --no-ff feature;
```

**检查冲突标记**：检测文件中的冲突标记
`git diff --check`

```bash
# 检查空白错误和冲突标记
git diff --check;
```

**放弃测试合并**：取消测试合并
`git merge --abort`

```bash
# 放弃测试合并
git merge --abort;
```
