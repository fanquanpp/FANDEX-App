# 撤销提交

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## revert 基本用法

**撤销单个提交**：创建反向提交撤销指定提交
`git revert <提交哈希>`

```bash
# 撤销 abc1234 提交
git revert abc1234;
```

**不自动提交**：将反向变更放入暂存区
`git revert -n <提交哈希>`

```bash
# 撤销 abc1234 但不自动提交
git revert -n abc1234;
```

**撤销连续多个提交**：撤销指定范围内的提交
`git revert <起始哈希>..<结束哈希>`

```bash
# 撤销 abc1234 到 def5678 之间的提交
git revert abc1234..def5678;
```

**撤销多个不连续提交**：批量撤销指定提交
`git revert <哈希1> <哈希2> <哈希3>`

```bash
# 撤销多个不连续的提交
git revert abc1234 def5678 ghi9012;
```

**指定 revert 消息**：自定义撤销提交的消息
`git revert -m "<消息>" <提交哈希>`

```bash
# 撤销 abc1234 并指定消息
git revert -m "revert: 回退认证功能" abc1234;
```

## 合并提交的 revert

**查看合并提交的父提交**：确定合并提交的父分支
`git cat-file -p <合并提交哈希>`

```bash
# 查看 abc1234 合并提交的父提交
git cat-file -p abc1234;
# parent def5678  第一个父提交（主分支）
# parent ghi9012  第二个父提交（合并分支）
```

**revert 保留第一个父提交**：撤销合并分支的变更
`git revert -m 1 <合并提交哈希>`

```bash
# 撤销合并提交，保留主分支的变更
git revert -m 1 abc1234;
```

**revert 保留第二个父提交**：撤销主分支的变更
`git revert -m 2 <合并提交哈希>`

```bash
# 撤销合并提交，保留合并分支的变更
git revert -m 2 abc1234;
```

## 重新合并已撤销的分支

**revert 之前的 revert**：恢复被撤销的合并
`git revert <revert提交哈希>`

```bash
# 1. 先 revert 合并提交
git revert -m 1 merge-commit;
# 2. 后续需要重新合并时，revert 那个 revert
git revert revert-commit;
```

**重新合并分支**：再次执行合并操作
`git merge <分支名>`

```bash
# revert revert 后重新合并 feature 分支
git merge feature;
```

## revert 冲突处理

**解决 revert 冲突**：手动解决后继续
`git revert --continue`

```bash
# 1. 触发 revert 冲突
git revert abc1234;
# 2. 手动解决冲突文件
# 3. 添加解决后的文件
git add .;
# 4. 继续 revert 流程
git revert --continue;
```

**放弃 revert**：取消 revert 操作
`git revert --abort`

```bash
# 放弃当前 revert 操作
git revert --abort;
```

## reset 撤销提交

**软回退**：撤销提交，保留修改在暂存区
`git reset --soft HEAD~<n>`

```bash
# 撤销最近一次提交，修改保留在暂存区
git reset --soft HEAD~1;
```

**混合回退**：撤销提交，保留修改在工作区
`git reset --mixed HEAD~<n>`

```bash
# 撤销最近一次提交，修改保留在工作区
git reset --mixed HEAD~1;
```

**硬回退**：撤销提交并丢弃所有修改
`git reset --hard HEAD~<n>`

```bash
# 撤销最近一次提交并丢弃修改
git reset --hard HEAD~1;
```

---

## 撤销工作区修改

**撤销单个文件修改**：恢复到暂存区或上次提交状态
`git checkout -- <file>`

```bash
# 撤销 src/index.js 的工作区修改
git checkout -- src/index.js;
```

**使用 restore 撤销**：Git 2.23+ 推荐写法
`git restore <file>`

```bash
# 撤销指定文件修改
git restore src/index.js;
```

---

## 撤销暂存

**取消暂存（保留修改）**：将文件移出暂存区
`git reset HEAD <file>`

```bash
# 将 src/index.js 移出暂存区
git reset HEAD src/index.js;
```

**使用 restore 撤销暂存**：Git 2.23+ 推荐写法
`git restore --staged <file>`

```bash
# 取消暂存但保留工作区修改
git restore --staged src/index.js;
```

## 实际场景

**回退已推送的功能**：安全撤销远程提交
`git revert <提交哈希>`

```bash
# 撤销已推送的 abc1234 提交
git revert abc1234;
git push origin main;
```

**回退整个发布**：撤销版本范围内的提交
`git revert <起始标签>..<结束标签>`

```bash
# 回退 v1.0.0 到 v1.1.0 之间的所有提交
git revert v1.0.0..v1.1.0;
git push origin main;
```

**安全撤销错误提交**：保留撤销原因说明
`git revert <错误提交哈希>`

```bash
# 撤销错误提交并说明原因
git revert wrong-commit;
git commit -m "revert: 回退错误提交，原因：...";
```
