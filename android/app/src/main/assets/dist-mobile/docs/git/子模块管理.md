# 子模块管理

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 添加子模块

**添加子模块**：将外部仓库作为子模块嵌入
`git submodule add <仓库地址> <路径>`

```bash
# 添加 shared-lib 作为子模块到 lib/shared
git submodule add https://github.com/user/shared-lib.git lib/shared;
```

**提交子模块添加**：将子模块配置提交到主仓库
`git commit -m "<消息>"`

```bash
# 提交子模块添加
git commit -m "feat: add shared-lib submodule";
```

---

## 克隆含子模块的仓库

**递归克隆**：克隆时自动初始化子模块
`git clone --recurse-submodules <仓库地址>`

```bash
# 克隆并递归初始化所有子模块
git clone --recurse-submodules https://github.com/user/main-repo.git;
```

**先克隆再初始化**：分步克隆和初始化子模块
`git submodule init`

```bash
# 1. 克隆主仓库
git clone https://github.com/user/main-repo.git;
cd main-repo;
# 2. 初始化子模块
git submodule init;
# 3. 更新子模块
git submodule update;
```

**一步到位初始化**：初始化并更新所有子模块
`git submodule update --init --recursive`

```bash
# 初始化并递归更新所有子模块
git submodule update --init --recursive;
```

## 更新子模块

**更新到最新提交**：将子模块更新到远程最新
`git submodule update --remote`

```bash
# 更新所有子模块到远程最新提交
git submodule update --remote;
```

**更新指定子模块**：仅更新指定子模块
`git submodule update --remote <路径>`

```bash
# 仅更新 lib/shared 子模块
git submodule update --remote lib/shared;
```

**更新并合并**：更新子模块并合并到当前分支
`git submodule update --remote --merge`

```bash
# 更新所有子模块并合并
git submodule update --remote --merge;
```

---

## 删除子模块

**取消注册子模块**：从配置中移除子模块
`git submodule deinit -f <路径>`

```bash
# 取消注册 lib/shared 子模块
git submodule deinit -f lib/shared;
```

**删除子模块 Git 数据**：移除子模块的 Git 目录
`rm -rf .git/modules/<路径>`

```bash
# 删除子模块的 Git 数据
rm -rf .git/modules/lib/shared;
```

**从 Git 中移除子模块**：删除子模块文件
`git rm -f <路径>`

```bash
# 从 Git 中移除子模块
git rm -f lib/shared;
```

**提交删除**：提交子模块删除操作
`git commit -m "<消息>"`

```bash
# 提交子模块删除
git commit -m "chore: remove shared-lib submodule";
```

## .gitmodules 配置文件

**配置文件格式**：子模块的配置文件结构
`[submodule "<名称>"]`

```ini
# .gitmodules 文件格式
[submodule "lib/shared"]
    path = lib/shared
    url = https://github.com/user/shared-lib.git
    branch = main
```

## 子模块分离 HEAD 处理

**切换到分支**：将子模块从分离 HEAD 切换到分支
`git checkout <分支名>`

```bash
# 1. 进入子模块目录
cd lib/shared;
# 2. 切换到 main 分支
git checkout main;
# 3. 拉取更新
git pull;
# 4. 返回主仓库
cd ../..;
# 5. 添加子模块更新
git add lib/shared;
# 6. 提交更新
git commit -m "chore: update submodule";
```

## 子模块脏状态处理

**忽略子模块修改**：配置忽略子模块的脏状态
`git config submodule.<路径>.ignore dirty`

```bash
# 忽略 lib/shared 子模块的修改
git config submodule.lib/shared.ignore dirty;
```

**强制更新子模块**：丢弃子模块的本地修改
`git submodule update --force`

```bash
# 强制更新所有子模块
git submodule update --force;
```

## 子模块冲突

**采用当前分支的子模块版本**：冲突时使用 ours
`git checkout --ours <子模块路径>`

```bash
# 采用当前分支的子模块版本
git checkout --ours lib/shared;
```

**采用合并分支的子模块版本**：冲突时使用 theirs
`git checkout --theirs <子模块路径>`

```bash
# 采用合并分支的子模块版本
git checkout --theirs lib/shared;
# 添加解决后的子模块
git add lib/shared;
```

## git subtree 替代方案

**添加 subtree**：将外部仓库合并到指定路径
`git subtree add --prefix=<路径> <仓库地址> <分支> --squash`

```bash
# 添加 shared-lib 到 lib/shared
git subtree add --prefix=lib/shared https://github.com/user/shared-lib.git main --squash;
```

**更新 subtree**：拉取外部仓库的更新
`git subtree pull --prefix=<路径> <仓库地址> <分支> --squash`

```bash
# 更新 lib/shared 的 subtree
git subtree pull --prefix=lib/shared https://github.com/user/shared-lib.git main --squash;
```
