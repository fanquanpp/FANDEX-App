# 脚注

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 基本语法

**脚注引用：在正文中标记脚注**
`<文本>[^<标识符>]`
```markdown
这是一段正文[^1]，其中包含脚注引用。
也可以使用描述性标签[^note]。
```

**脚注定义：定义脚注内容**
`[^<标识符>]: <脚注内容>`
```markdown
[^1]: 这是第一个脚注的内容。
[^note]: 这是描述性标签的脚注内容。
```

---

## 脚注标签类型

**数字标签：顺序引用**
`[^<数字>]`
```markdown
正文内容[^1]。
[^1]: 脚注内容。
```

**描述标签：语义化引用**
`[^<描述>]`
```markdown
正文内容[^source]。
[^source]: 这是来源说明。
```

---

## 多行脚注

**多行脚注：使用缩进延续**
`[^<标识符>]: <第一行>\n    <第二行>`
```markdown
[^long]:
    这是第一行。
    这是第二行（缩进4个空格）。
    这是第三行。
```

---

## 多次引用同一脚注

**重复引用：同一标识符可多次使用**
`<文本>[^<标识符>] ... <文本>[^<标识符>]`
```markdown
TypeScript[^ts] 是 JavaScript 的超集。
TypeScript[^ts] 添加了静态类型检查。

[^ts]: TypeScript 由 Microsoft 开发，2012 年首次发布。
```

---

## 脚注中的 Markdown

**脚注支持行内 Markdown 语法**
`[^<标识符>]: <Markdown 内容>`
```markdown
正文引用[^api]。

[^api]: 参见 [API 文档](https://example.com/api) 中关于 `fetch()` 方法的说明。
```

---

## 行内脚注

**Obsidian 行内脚注：无需单独定义**
`<文本>^[<脚注内容>]`
```markdown
正文内容^[这是行内脚注，无需单独定义。]
```

---

## 替代方案

**行内链接替代：不支持脚注时使用**
`<文本>[<编号>]\n[<编号>]: <引用内容>`
```markdown
根据研究[1]，结论如下。

[1] Smith, J. "Deep Learning Survey." 2023.
```

**HTML 锚点替代**
`<a href="#<锚点>">[<编号>]</a>`
```markdown
根据研究<a href="#ref1">[1]</a>。

<a name="ref1"></a>
1. Smith, J. "Deep Learning Survey." 2023.
```
