# 内存淘汰策略

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 触发条件

**CONFIG SET maxmemory：设置最大内存**
`CONFIG SET maxmemory <bytes>`
```bash
# 设置最大内存
CONFIG SET maxmemory 4gb
```

**INFO memory：查看内存使用**
`INFO memory`
```bash
# 查看当前内存使用
INFO memory
# used_memory: 3.8GB
# maxmemory: 4GB
```

---

## 淘汰策略配置

**CONFIG SET maxmemory-policy：设置淘汰策略**
`CONFIG SET maxmemory-policy <policy>`
```bash
# 八种淘汰策略:
# noeviction      - 不淘汰
# allkeys-lru     - 所有键 LRU
# allkeys-lfu     - 所有键 LFU
# allkeys-random  - 所有键随机
# volatile-lru    - 有TTL的键 LRU
# volatile-lfu    - 有TTL的键 LFU
# volatile-random - 有TTL的键随机
# volatile-ttl    - 有TTL的键最短TTL优先

CONFIG SET maxmemory-policy allkeys-lfu
```

## LRU 算法

**近似 LRU：采样淘汰最久未访问的键**
`maxmemory-samples <N>`
```bash
# 采样数配置
CONFIG SET maxmemory-samples 5
# 采样数对效果的影响:
# maxmemory-samples = 3:  接近真实LRU的 80%
# maxmemory-samples = 5:  接近真实LRU的 90%  ← 默认
# maxmemory-samples = 10: 接近真实LRU的 95%
# maxmemory-samples = 20: 接近真实LRU的 98%
```

**redisObject 结构：LRU 时钟字段**
`struct redisObject { unsigned lru:24; }`
```c
// 每个 Redis 对象头包含一个 24 位的 LRU 时钟
typedef struct redisObject {
    unsigned type:4;
    unsigned encoding:4;
    unsigned lru:24;    // LRU 时间戳（精度约1.5分钟）
    int refcount;
    void *ptr;
} robj;

// LRU 时钟精度: 1000ms / 300 ≈ 3.3秒
// 24位最大值: 2^24 = 16777216 ≈ 194天循环
// 计算空闲时间: current_lru - object.lru
```

## LFU 算法

**LFU 配置：设置衰减时间与计数器因子**
`CONFIG SET lfu-decay-time <minutes> | CONFIG SET lfu-log-factor <factor>`
```bash
# 淘汰策略
CONFIG SET maxmemory-policy allkeys-lfu

# 衰减时间（分钟）
CONFIG SET lfu-decay-time 1

# 计数器初始值
CONFIG SET lfu-log-factor 10
```

**LFU 计数器更新：对数计数器源码**
`uint8_t LFULogIncr(uint8_t counter)`
```c
// 24位 lru 字段:
//   高16位: 最后衰减时间（分钟级）
//   低8位:  对数计数器（logarithmic counter）
// 计数器范围: 0-255
// 实际频率范围: 1-约100万次/分钟

uint8_t LFULogIncr(uint8_t counter) {
    if (counter == 255) return 255;
    double r = (double)rand() / RAND_MAX;
    double baseval = counter - LFU_INIT_VAL;  // LFU_INIT_VAL = 5
    if (baseval < 0) baseval = 0;
    double p = 1.0 / (baseval * 10 + 1);  // 概率递减
    if (r < p) counter++;
    return counter;
}
```

## 监控与调优

**INFO stats：查看淘汰统计**
`INFO stats`
```bash
# 查看淘汰统计
INFO stats
# evicted_keys: 1234  ← 被淘汰的键数量

# 调优建议:
# 1. evicted_keys 持续增长 → 增大 maxmemory 或优化策略
# 2. 缓存命中率低 → 考虑换策略（LRU → LFU）
# 3. 内存碎片率高 → 重启或使用 activedefrag
```
