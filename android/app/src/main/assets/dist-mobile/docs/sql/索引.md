# 索引

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 创建索引

**CREATE INDEX：创建普通索引**
`CREATE [UNIQUE] INDEX <index_name> ON <table>(<列> [, ...])`
```sql
-- 单列索引
CREATE INDEX idx_name ON employees(name);

-- 唯一索引
CREATE UNIQUE INDEX idx_email ON employees(email);

-- 复合索引
CREATE INDEX idx_dept_salary ON employees(dept_id, salary);
```

**CREATE TABLE 中定义索引**
`INDEX <name> (<列>) | KEY <name> (<列>)`
```sql
-- MySQL: 在 CREATE TABLE 中创建索引
CREATE TABLE employees (
  id BIGINT PRIMARY KEY,
  name VARCHAR(50),
  email VARCHAR(100),
  dept_id BIGINT,
  INDEX idx_name (name),
  UNIQUE INDEX idx_email (email),
  INDEX idx_dept (dept_id)
);
```

## 索引类型

**B-Tree 索引：默认索引类型**
`CREATE INDEX <name> ON <table>(<列>) USING BTREE`
```sql
-- 默认 B-Tree 索引
CREATE INDEX idx_name ON employees(name);
-- 显式指定
CREATE INDEX idx_name ON employees USING BTREE (name);
```

**Hash 索引：哈希索引**
`CREATE INDEX <name> ON <table> USING HASH (<列>)`
```sql
-- PostgreSQL: Hash 索引
CREATE INDEX idx_hash ON users USING HASH (email);

-- MySQL: MEMORY 引擎支持 Hash
-- InnoDB 自适应哈希索引（自动管理）
```

**GiST / GIN 索引：PostgreSQL 扩展索引**
`CREATE INDEX <name> ON <table> USING GIST|GIN (<列>)`
```sql
-- GIN 索引（适合数组、全文搜索）
CREATE INDEX idx_tags ON posts USING GIN (tags);

-- GiST 索引（适合地理空间）
CREATE INDEX idx_location ON stores USING GIST (location);
```

## 复合索引

**复合索引：多列索引**
`CREATE INDEX <name> ON <table>(<列1>, <列2>, ...)`
```sql
-- 复合索引
CREATE INDEX idx_dept_salary ON employees(dept_id, salary);

-- 最左前缀原则:
-- WHERE dept_id = 1                    -- 命中索引
-- WHERE dept_id = 1 AND salary > 50000 -- 命中索引
-- WHERE salary > 50000                  -- 不命中索引（跳过了 dept_id）
```

## 函数索引

**函数索引：基于表达式创建索引**
`CREATE INDEX <name> ON <table>(<函数>(<列>))`
```sql
-- PostgreSQL: 函数索引
CREATE INDEX idx_lower_email ON users(LOWER(email));

-- MySQL 8.0+: 函数索引
CREATE INDEX idx_lower_email ON users((LOWER(email)));

-- 查询时使用
SELECT * FROM users WHERE LOWER(email) = 'test@example.com';
```

## 前缀索引

**前缀索引：对字符串前 N 个字符建索引**
`CREATE INDEX <name> ON <table>(<列>(<N>))`
```sql
-- MySQL: 前缀索引
CREATE INDEX idx_name_prefix ON employees(name(10));

-- 选择前缀长度
SELECT
  COUNT(DISTINCT LEFT(name, 5)) / COUNT(*) AS sel_5,
  COUNT(DISTINCT LEFT(name, 10)) / COUNT(*) AS sel_10,
  COUNT(DISTINCT LEFT(name, 15)) / COUNT(*) AS sel_15
FROM employees;
```

## 覆盖索引

**覆盖索引：索引包含查询所需所有列**
`CREATE INDEX <name> ON <table>(<列1>, <列2>, ...) INCLUDE (<列3>)`
```sql
-- PostgreSQL: INCLUDE 子句
CREATE INDEX idx_covering ON employees(dept_id) INCLUDE (name, salary);

-- MySQL: 复合索引作为覆盖索引
CREATE INDEX idx_covering ON employees(dept_id, name, salary);

-- 查询使用覆盖索引（不需要回表）
SELECT dept_id, name, salary FROM employees WHERE dept_id = 1;
```

## 索引管理

**查看索引**
`SHOW INDEX FROM <table> | \d <table>`
```sql
-- MySQL: 查看索引
SHOW INDEX FROM employees;

-- PostgreSQL: 查看索引
\d employees

-- 查询所有索引
SELECT * FROM pg_indexes WHERE tablename = 'employees';
```

**删除索引**
`DROP INDEX <index_name> | DROP INDEX <index_name> ON <table>`
```sql
-- PostgreSQL / SQL Server
DROP INDEX idx_name;

-- MySQL
DROP INDEX idx_name ON employees;
```

**重建索引**
`REINDEX INDEX <name> | ALTER INDEX <name> REBUILD`
```sql
-- PostgreSQL
REINDEX INDEX idx_name;
REINDEX TABLE employees;

-- SQL Server
ALTER INDEX idx_name ON employees REBUILD;

-- MySQL: 无直接重建命令，使用 DROP + CREATE
DROP INDEX idx_name ON employees;
CREATE INDEX idx_name ON employees(name);
```

## EXPLAIN 执行计划

**EXPLAIN：查看查询执行计划**
`EXPLAIN <SELECT 语句>`
```sql
-- MySQL: EXPLAIN
EXPLAIN SELECT * FROM employees WHERE dept_id = 1;

-- MySQL EXPLAIN 输出关键字段:
-- type: 访问类型（system > const > eq_ref > ref > range > index > ALL）
-- key: 使用的索引
-- rows: 预估扫描行数
-- Extra: Using index（覆盖索引）、Using filesort、Using temporary

-- PostgreSQL: EXPLAIN ANALYZE
EXPLAIN ANALYZE SELECT * FROM employees WHERE dept_id = 1;
```

## 索引优化建议

**索引使用建议**
`CREATE INDEX <name> ON <table>(<列>)`
```sql
-- 1. WHERE 条件列建索引
CREATE INDEX idx_status ON orders(status);

-- 2. JOIN 连接列建索引
CREATE INDEX idx_orders_user_id ON orders(user_id);

-- 3. ORDER BY / GROUP BY 列建索引
CREATE INDEX idx_created_at ON orders(created_at);

-- 4. 避免在索引列上使用函数
-- 不推荐: WHERE YEAR(created_at) = 2026
-- 推荐: WHERE created_at >= '2026-01-01' AND created_at < '2027-01-01'

-- 5. 避免在低基数列建索引
-- 不推荐: gender（只有 M/F）
```
