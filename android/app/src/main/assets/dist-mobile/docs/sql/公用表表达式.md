# 公用表表达式

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 基本 CTE

**WITH：定义公用表表达式**
`WITH <cte_name> AS (SELECT ...) SELECT ... FROM <cte_name>`
```sql
-- 基本 CTE
WITH dept_avg AS (
  SELECT dept_id, AVG(salary) AS avg_salary
  FROM employees
  GROUP BY dept_id
)
SELECT e.name, e.salary, d.avg_salary
FROM employees e
JOIN dept_avg d ON e.dept_id = d.dept_id
WHERE e.salary > d.avg_salary;
```

**多个 CTE：定义多个临时结果集**
`WITH <cte1> AS (...), <cte2> AS (...) SELECT ...`
```sql
-- 多个 CTE（逗号分隔）
WITH
  dept_avg AS (
    SELECT dept_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY dept_id
  ),
  dept_count AS (
    SELECT dept_id, COUNT(*) AS emp_count
    FROM employees
    GROUP BY dept_id
  )
SELECT
  d.dept_name,
  a.avg_salary,
  c.emp_count
FROM departments d
JOIN dept_avg a ON d.id = a.dept_id
JOIN dept_count c ON d.id = c.dept_id;
```

**CTE 引用其他 CTE**
`WITH <cte1> AS (...), <cte2> AS (... FROM <cte1> ...) SELECT ...`
```sql
-- CTE 引用前面的 CTE
WITH
  base AS (
    SELECT dept_id, salary FROM employees WHERE status = 'active'
  ),
  dept_stats AS (
    SELECT dept_id, AVG(salary) AS avg_salary, COUNT(*) AS cnt
    FROM base
    GROUP BY dept_id
  )
SELECT * FROM dept_stats WHERE avg_salary > 50000;
```

## 递归 CTE

**递归 CTE：处理层级数据**
`WITH RECURSIVE <cte> AS (<非递归部分> UNION [ALL] <递归部分>) SELECT ...`
```sql
-- 递归 CTE: 查询组织架构
WITH RECURSIVE org_tree AS (
  -- 基础查询（非递归部分）
  SELECT id, name, manager_id, 1 AS level
  FROM employees
  WHERE manager_id IS NULL

  UNION ALL

  -- 递归部分
  SELECT e.id, e.name, e.manager_id, t.level + 1
  FROM employees e
  JOIN org_tree t ON e.manager_id = t.id
)
SELECT * FROM org_tree ORDER BY level, name;
```

**递归 CTE 查询子树**
`WITH RECURSIVE <cte> AS (...) SELECT ...`
```sql
-- 查询某个节点的所有子节点
WITH RECURSIVE sub_tree AS (
  -- 起点
  SELECT id, name, parent_id, 1 AS depth
  FROM categories
  WHERE id = 5  -- 从 id=5 开始

  UNION ALL

  -- 递归
  SELECT c.id, c.name, c.parent_id, t.depth + 1
  FROM categories c
  JOIN sub_tree t ON c.parent_id = t.id
)
SELECT * FROM sub_tree;
```

**递归 CTE 查询路径**
`WITH RECURSIVE <cte> AS (... <path列> ...) SELECT ...`
```sql
-- 查询路径
WITH RECURSIVE path_cte AS (
  SELECT id, name, parent_id, name::TEXT AS path
  FROM categories
  WHERE parent_id IS NULL

  UNION ALL

  SELECT c.id, c.name, c.parent_id, t.path || ' > ' || c.name
  FROM categories c
  JOIN path_cte t ON c.parent_id = t.id
)
SELECT id, path FROM path_cte;
```

## CTE 与 DML

**CTE 用于 INSERT**
`WITH <cte> AS (...) INSERT INTO <table> SELECT ... FROM <cte>`
```sql
-- CTE 用于 INSERT
WITH active_users AS (
  SELECT * FROM users WHERE status = 'active'
)
INSERT INTO user_backup
SELECT * FROM active_users;
```

**CTE 用于 UPDATE**
`WITH <cte> AS (...) UPDATE <table> SET ... FROM <cte> WHERE ...`
```sql
-- PostgreSQL: CTE 用于 UPDATE
WITH high_salary AS (
  SELECT id FROM employees WHERE salary > 100000
)
UPDATE employees
SET bonus = salary * 0.1
WHERE id IN (SELECT id FROM high_salary);
```

**CTE 用于 DELETE**
`WITH <cte> AS (...) DELETE FROM <table> WHERE <列> IN (SELECT ... FROM <cte>)`
```sql
-- CTE 用于 DELETE
WITH inactive_users AS (
  SELECT id FROM users WHERE last_login < '2024-01-01'
)
DELETE FROM users
WHERE id IN (SELECT id FROM inactive_users);
```

## CTE 与窗口函数

**CTE 结合窗口函数**
`WITH <cte> AS (SELECT ... <窗口函数> ...) SELECT ...`
```sql
-- CTE 结合窗口函数
WITH ranked_employees AS (
  SELECT
    name,
    department,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank_num
  FROM employees
)
SELECT * FROM ranked_employees WHERE rank_num <= 3;
```

## 递归 CTE 终止条件

**防止无限递归**
`WITH RECURSIVE <cte> AS (... WHERE <depth> < <N>) SELECT ...`
```sql
-- 限制递归深度
WITH RECURSIVE org_tree AS (
  SELECT id, name, manager_id, 1 AS level
  FROM employees
  WHERE manager_id IS NULL

  UNION ALL

  SELECT e.id, e.name, e.manager_id, t.level + 1
  FROM employees e
  JOIN org_tree t ON e.manager_id = t.id
  WHERE t.level < 10  -- 限制递归深度
)
SELECT * FROM org_tree;

-- MySQL: 全局限制
SET SESSION cte_max_recursion_depth = 100;

-- PostgreSQL: 全局限制
-- default: 100（通过 max_recursive_depth 或 standard_conforming_strings 控制）
```
