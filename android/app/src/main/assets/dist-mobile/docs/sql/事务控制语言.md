# 事务控制语言

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 事务基本操作

**BEGIN：开始事务**
`BEGIN | BEGIN TRANSACTION | START TRANSACTION`
```sql
-- 开始事务
BEGIN;

-- 等价写法
BEGIN TRANSACTION;
START TRANSACTION;
```

**COMMIT：提交事务**
`COMMIT | COMMIT TRANSACTION`
```sql
-- 提交事务
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
```

**ROLLBACK：回滚事务**
`ROLLBACK | ROLLBACK TRANSACTION`
```sql
-- 回滚事务
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
-- 发现错误，回滚
ROLLBACK;
```

## 自动提交

**查看自动提交状态**
`SHOW autocommit`
```sql
-- MySQL: 查看自动提交
SHOW VARIABLES LIKE 'autocommit';

-- PostgreSQL: 查看自动提交
SHOW autocommit;
```

**设置自动提交**
`SET autocommit = 0 | 1`
```sql
-- MySQL: 关闭自动提交
SET autocommit = 0;

-- MySQL: 开启自动提交（默认）
SET autocommit = 1;
```

## 保存点

**SAVEPOINT：设置保存点**
`SAVEPOINT <savepoint_name>`
```sql
-- 设置保存点
BEGIN;
INSERT INTO orders (user_id, amount) VALUES (1, 100);
SAVEPOINT after_insert;
UPDATE users SET balance = balance - 100 WHERE id = 1;
```

**ROLLBACK TO：回滚到保存点**
`ROLLBACK TO [SAVEPOINT] <savepoint_name>`
```sql
-- 回滚到保存点（保存点之前的操作保留）
ROLLBACK TO after_insert;
```

**RELEASE SAVEPOINT：释放保存点**
`RELEASE SAVEPOINT <savepoint_name>`
```sql
-- 释放保存点（删除保存点，之后的 ROLLBACK TO 不能使用）
RELEASE SAVEPOINT after_insert;
```

## 隔离级别

**查看隔离级别**
`SHOW TRANSACTION ISOLATION LEVEL`
```sql
-- MySQL: 查看隔离级别
SELECT @@transaction_isolation;

-- PostgreSQL: 查看隔离级别
SHOW transaction_isolation;
```

**设置隔离级别**
`SET TRANSACTION ISOLATION LEVEL <level>`
```sql
-- MySQL: 设置隔离级别
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;
SET GLOBAL TRANSACTION ISOLATION LEVEL REPEATABLE READ;

-- PostgreSQL: 设置隔离级别
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

-- 在 BEGIN 后设置
BEGIN;
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SELECT * FROM accounts;
COMMIT;
```

**四种隔离级别**
`READ UNCOMMITTED | READ COMMITTED | REPEATABLE READ | SERIALIZABLE`
```sql
-- READ UNCOMMITTED: 读未提交（脏读）
SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;

-- READ COMMITTED: 读已提交（避免脏读，PostgreSQL/Oracle 默认）
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

-- REPEATABLE READ: 可重复读（避免不可重复读，MySQL 默认）
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;

-- SERIALIZABLE: 串行化（避免幻读，最高隔离级别）
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
```

## 锁

**SELECT ... FOR UPDATE：行级排他锁**
`SELECT ... FROM <table> WHERE <condition> FOR UPDATE`
```sql
-- 行级排他锁（写锁）
BEGIN;
SELECT * FROM accounts WHERE id = 1 FOR UPDATE;
-- 其他事务的 FOR UPDATE 会被阻塞
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
COMMIT;
```

**SELECT ... FOR SHARE：行级共享锁**
`SELECT ... FROM <table> WHERE <condition> FOR SHARE`
```sql
-- 行级共享锁（读锁）
BEGIN;
SELECT * FROM products WHERE id = 1 FOR SHARE;
-- 其他事务可以读，但不能 FOR UPDATE
COMMIT;
```

**LOCK TABLE：表级锁**
`LOCK TABLE <table> IN <mode> MODE`
```sql
-- PostgreSQL: 表级锁
LOCK TABLE users IN EXCLUSIVE MODE;

-- MySQL: 表级锁
LOCK TABLES users WRITE;
UNLOCK TABLES;
```

```sql
-- 3. 使用合适的隔离级别
```

## 事务特性

**ACID 特性验证**
`BEGIN ... COMMIT | ROLLBACK`
```sql
-- 原子性（Atomicity）: 事务内操作要么全部成功，要么全部回滚
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;  -- 或 ROLLBACK

-- 一致性（Consistency）: 事务前后数据一致
-- 总金额不变: 100 + 200 = 300 → 0 + 300 = 300

-- 隔离性（Isolation）: 并发事务互不干扰
-- 通过隔离级别控制

-- 持久性（Durability）: 提交后数据永久保存
-- 通过 WAL/redo log 保证
```
