# 连接查询

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 连接基本语法

**JOIN 基本语法：组合多表数据**
`SELECT <select_list> FROM <left_table> [AS] <alias> [JOIN_TYPE] <right_table> [AS] <alias> ON <join_condition>`
```sql
SELECT select_list
FROM left_table [AS] alias
[JOIN_TYPE] right_table [AS] alias
ON join_condition;
```

---

## INNER JOIN

**INNER JOIN：返回两表中匹配的行**
`FROM <left_table> INNER JOIN <right_table> ON <condition>`
```sql
-- 只返回两表中满足连接条件的行
SELECT e.name, d.dept_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.id;

-- INNER 可省略
SELECT e.name, d.dept_name
FROM employees e
JOIN departments d ON e.dept_id = d.id;
```

**非等值连接：使用非等号条件**
`FROM <left_table> JOIN <right_table> ON <非等值条件>`
```sql
-- 等值连接（最常见）
SELECT e.name, d.dept_name
FROM employees e
JOIN departments d ON e.dept_id = d.id;

-- 非等值连接
SELECT e.name, g.grade
FROM employees e
JOIN salary_grades g ON e.salary BETWEEN g.min_salary AND g.max_salary;
```

**多表连接：连接三张及以上表**
`FROM <t1> JOIN <t2> ON ... JOIN <t3> ON ...`
```sql
SELECT e.name, d.dept_name, j.job_title
FROM employees e
JOIN departments d ON e.dept_id = d.id
JOIN jobs j ON e.job_id = j.id
WHERE d.region = 'East';
```

## LEFT JOIN

**LEFT JOIN：返回左表全部行**
`FROM <left_table> LEFT JOIN <right_table> ON <condition>`
```sql
-- 返回左表所有行，右表无匹配时填 NULL
SELECT d.dept_name, e.name
FROM departments d
LEFT JOIN employees e ON d.id = e.dept_id;
```

**LEFT JOIN 查找无匹配行**
`FROM <left_table> LEFT JOIN <right_table> ON <condition> WHERE <right_table>.<col> IS NULL`
```sql
-- 场景1：查找没有员工的部门
SELECT d.dept_name
FROM departments d
LEFT JOIN employees e ON d.id = e.dept_id
WHERE e.id IS NULL;

-- 场景2：统计每个部门的员工数（包括0人部门）
SELECT d.dept_name, COUNT(e.id) AS emp_count
FROM departments d
LEFT JOIN employees e ON d.id = e.dept_id
GROUP BY d.id, d.dept_name;
```

**LEFT JOIN + WHERE 陷阱：右表过滤条件应放 ON 子句**
`FROM <left_table> LEFT JOIN <right_table> ON <condition> AND <右表过滤>`
```sql
-- 错误：WHERE 条件使 LEFT JOIN 退化为 INNER JOIN
SELECT d.dept_name, e.name
FROM departments d
LEFT JOIN employees e ON d.id = e.dept_id
WHERE e.status = 'active';  -- 过滤掉了没有员工的部门

-- 正确：将右表过滤条件移到 ON 子句
SELECT d.dept_name, e.name
FROM departments d
LEFT JOIN employees e ON d.id = e.dept_id AND e.status = 'active';
```

## RIGHT JOIN

**RIGHT JOIN：返回右表全部行**
`FROM <left_table> RIGHT JOIN <right_table> ON <condition>`
```sql
-- 返回右表所有行，左表无匹配时填 NULL
SELECT e.name, d.dept_name
FROM employees e
RIGHT JOIN departments d ON e.dept_id = d.id;

-- 等价写法（推荐使用 LEFT JOIN）
SELECT e.name, d.dept_name
FROM departments d
LEFT JOIN employees e ON e.dept_id = d.id;
```

## FULL JOIN

**FULL JOIN：返回两表所有行**
`FROM <left_table> FULL JOIN <right_table> ON <condition>`
```sql
-- 返回两表所有行，不匹配时填 NULL
SELECT e.name, d.dept_name
FROM employees e
FULL JOIN departments d ON e.dept_id = d.id;
```

**FULL JOIN 查找不匹配行**
`FROM <left_table> FULL JOIN <right_table> ON <condition> WHERE <左表>.<id> IS NULL OR <右表>.<id> IS NULL`
```sql
-- 查找两表不匹配的行
SELECT e.name, d.dept_name
FROM employees e
FULL JOIN departments d ON e.dept_id = d.id
WHERE e.id IS NULL OR d.id IS NULL;
```

**MySQL FULL JOIN 替代：UNION ALL**
`LEFT JOIN ... UNION ALL RIGHT JOIN ... WHERE IS NULL`
```sql
-- MySQL 不支持 FULL JOIN，使用 UNION ALL 替代
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.id
UNION ALL
SELECT e.name, d.dept_name
FROM employees e
RIGHT JOIN departments d ON e.dept_id = d.id
WHERE e.id IS NULL;
```

## CROSS JOIN

**CROSS JOIN：笛卡尔积**
`FROM <left_table> CROSS JOIN <right_table>`
```sql
-- 笛卡尔积：m行 × n行 = m×n行
SELECT d.dept_name, j.job_title
FROM departments d
CROSS JOIN jobs j;

-- 隐式交叉连接
SELECT d.dept_name, j.job_title
FROM departments d, jobs j;
```

## 自连接

**自连接：表与自身连接**
`FROM <table> AS <alias1> JOIN <table> AS <alias2> ON <condition>`
```sql
-- 员工与经理关系
SELECT
  e.name AS employee,
  m.name AS manager
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;

-- 查找同一部门中薪资相同的员工
SELECT a.name, b.name, a.salary
FROM employees a
JOIN employees b ON a.dept_id = b.dept_id AND a.salary = b.salary AND a.id < b.id;
```

---

## USING 子句

**USING：指定同名列连接**
`FROM <left_table> JOIN <right_table> USING (<column>)`
```sql
-- USING 子句: 指定同名列连接（比 ON 更简洁）
SELECT e.name, department_id
FROM employees e
JOIN departments d USING (department_id);

-- NATURAL JOIN: 自动按同名列连接（不推荐，不可控）
SELECT * FROM employees NATURAL JOIN departments;
```

## 连接条件与过滤条件

**ON 与 WHERE 的区别**
`ON <连接条件> | WHERE <过滤条件>`
```sql
-- INNER JOIN 中 ON 和 WHERE 等价（逻辑上）
SELECT * FROM a INNER JOIN b ON a.id = b.a_id AND a.status = 'active';
-- 等价于
SELECT * FROM a INNER JOIN b ON a.id = b.a_id WHERE a.status = 'active';

-- OUTER JOIN 中 ON 和 WHERE 不等价
SELECT * FROM a LEFT JOIN b ON a.id = b.a_id AND a.status = 'active';
-- a.status = 'active' 只影响右表匹配，左表行仍保留

SELECT * FROM a LEFT JOIN b ON a.id = b.a_id WHERE a.status = 'active';
-- a.status = 'active' 过滤最终结果，左表不满足的行被移除
```
