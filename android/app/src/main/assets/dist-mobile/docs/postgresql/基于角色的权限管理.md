# PostgreSQL 基于角色的权限管理

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 角色创建

**创建登录角色（用户）**
`CREATE ROLE <角色名> LOGIN PASSWORD '<密码>'`
```sql
-- 创建登录角色
CREATE ROLE app_user LOGIN PASSWORD 'password';
```

**创建组角色**
`CREATE ROLE <角色名> NOLOGIN`
```sql
-- 创建组角色（不能登录）
CREATE ROLE readonly NOLOGIN;
CREATE ROLE readwrite NOLOGIN;
CREATE ROLE admin NOLOGIN;
```

---

## 权限授予

**授予表权限**
`GRANT <权限列表> ON ALL TABLES IN SCHEMA <模式名> TO <角色名>`
```sql
-- 授予只读权限
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;

-- 授予读写权限
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO readwrite;

-- 授予所有权限
GRANT ALL PRIVILEGES ON SCHEMA public TO admin;
```

**将组角色授予用户**
`GRANT <组角色名> TO <用户角色名>`
```sql
-- 分配组角色给用户
GRANT readonly TO app_read;
GRANT readwrite TO app_write;
```

---

## 角色继承

**切换角色**
`SET ROLE <角色名>`
```sql
-- 切换到 readonly 角色
SET ROLE readonly;
SELECT current_user;
```

**恢复角色**
`RESET ROLE`
```sql
-- 恢复原角色
RESET ROLE;
```

**禁止角色继承**
`ALTER ROLE <角色名> NOINHERIT`
```sql
-- 禁止继承，需要显式 SET ROLE
ALTER ROLE app_user NOINHERIT;
```

---

## 默认权限

**设置新建表的默认权限**
`ALTER DEFAULT PRIVILEGES IN SCHEMA <模式名> GRANT <权限> ON TABLES TO <角色名>`
```sql
-- 新建表的默认权限
ALTER DEFAULT PRIVILEGES IN SCHEMA public
    GRANT SELECT ON TABLES TO readonly;
```

**指定授权人**
`ALTER DEFAULT PRIVILEGES FOR ROLE <角色名> GRANT <权限> ON TABLES TO <角色名>`
```sql
-- 指定 admin 角色创建的表的默认权限
ALTER DEFAULT PRIVILEGES FOR ROLE admin
    GRANT SELECT ON TABLES TO readonly;
```

## 查看权限

**查看表权限**
`SELECT grantee, privilege_type FROM information_schema.table_privileges WHERE table_name = '<表名>'`
```sql
-- 查看表权限
SELECT grantee, privilege_type
FROM information_schema.table_privileges
WHERE table_name = 'employees';
```

**查看角色成员**
`SELECT r.rolname, m.rolname AS member FROM pg_roles r JOIN pg_auth_members am ON r.oid = am.roleid JOIN pg_roles m ON am.member = m.oid`
```sql
-- 查看角色成员关系
SELECT r.rolname, m.rolname AS member
FROM pg_roles r
JOIN pg_auth_members am ON r.oid = am.roleid
JOIN pg_roles m ON am.member = m.oid;
```
