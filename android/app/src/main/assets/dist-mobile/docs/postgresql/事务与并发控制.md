# PostgreSQL 事务与并发控制

> **符号约定**：`< >` 必填参数 | `[ ]` 可选参数

---

## 事务控制

**开启事务**
`BEGIN [ISOLATION LEVEL <隔离级别>]`
```sql
-- 开启事务
BEGIN;

-- 指定隔离级别
BEGIN ISOLATION LEVEL READ COMMITTED;
BEGIN ISOLATION LEVEL REPEATABLE READ;
BEGIN ISOLATION LEVEL SERIALIZABLE;
```

**提交事务**
`COMMIT`
```sql
-- 提交事务
COMMIT;
```

**回滚事务**
`ROLLBACK`
```sql
-- 回滚事务
ROLLBACK;
```

**保存点**
`SAVEPOINT <保存点名>` / `ROLLBACK TO <保存点名>`
```sql
-- 使用保存点
BEGIN;
INSERT INTO accounts (name, balance) VALUES ('Alice', 1000);
SAVEPOINT sp1;
INSERT INTO accounts (name, balance) VALUES ('Bob', 500);
ROLLBACK TO sp1;
COMMIT;
```

## 事务隔离级别示例

**读已提交（Read Committed）**
`BEGIN ISOLATION LEVEL READ COMMITTED`
```sql
-- 会话1：读已提交
BEGIN ISOLATION LEVEL READ COMMITTED;
SELECT balance FROM accounts WHERE name = 'Alice';

-- 会话2：更新并提交
BEGIN;
UPDATE accounts SET balance = 600 WHERE name = 'Alice';
COMMIT;

-- 会话1 再次查询（看到新数据）
SELECT balance FROM accounts WHERE name = 'Alice';
COMMIT;
```

**可重复读（Repeatable Read）**
`BEGIN ISOLATION LEVEL REPEATABLE READ`
```sql
-- 会话1：可重复读
BEGIN ISOLATION LEVEL REPEATABLE READ;
SELECT balance FROM accounts WHERE name = 'Alice';

-- 会话2：更新并提交
BEGIN;
UPDATE accounts SET balance = 400 WHERE name = 'Alice';
COMMIT;

-- 会话1 再次查询（仍看到旧数据）
SELECT balance FROM accounts WHERE name = 'Alice';
COMMIT;
```

**可序列化（Serializable）**
`BEGIN ISOLATION LEVEL SERIALIZABLE`
```sql
-- 会话1：可序列化
BEGIN ISOLATION LEVEL SERIALIZABLE;
SELECT COUNT(*) FROM accounts WHERE balance > 500;

-- 会话2：插入并提交
BEGIN ISOLATION LEVEL SERIALIZABLE;
INSERT INTO accounts (name, balance) VALUES ('Bob', 600);
COMMIT;

-- 会话1 提交（可能抛出序列化错误）
INSERT INTO audit_log SELECT 'high_balance', COUNT(*)
  FROM accounts WHERE balance > 500;
COMMIT;
```

## 表级锁

**显式获取表锁**
`LOCK TABLE <表名> IN <锁模式> MODE`
```sql
-- 获取排他锁
LOCK TABLE accounts IN ACCESS EXCLUSIVE MODE;
```

**查看当前锁**
`SELECT locktype, relation::regclass, mode, pid, granted FROM pg_locks WHERE relation IS NOT NULL ORDER BY relation, mode`
```sql
-- 查看锁信息
SELECT locktype, relation::regclass, mode, pid, granted
FROM pg_locks
WHERE relation IS NOT NULL
ORDER BY relation, mode;
```

## 行级锁

**FOR UPDATE**
`SELECT * FROM <表名> WHERE <条件> FOR UPDATE`
```sql
-- 排他行锁（更新/删除时自动获取）
SELECT * FROM accounts WHERE id = 1 FOR UPDATE;
```

**FOR NO KEY UPDATE**
`SELECT * FROM <表名> WHERE <条件> FOR NO KEY UPDATE`
```sql
-- 弱于 FOR UPDATE，不阻塞 KEY SELECT
SELECT * FROM accounts WHERE id = 1 FOR NO KEY UPDATE;
```

**FOR SHARE**
`SELECT * FROM <表名> WHERE <条件> FOR SHARE`
```sql
-- 共享行锁（允许并发读，阻止更新）
SELECT * FROM accounts WHERE id = 1 FOR SHARE;
```

**FOR KEY SHARE**
`SELECT * FROM <表名> WHERE <条件> FOR KEY SHARE`
```sql
-- 弱于 FOR SHARE，仅锁键列
SELECT * FROM accounts WHERE id = 1 FOR KEY SHARE;
```

**NOWAIT**
`SELECT * FROM <表名> WHERE <条件> FOR UPDATE NOWAIT`
```sql
-- 不等待锁，立即报错
SELECT * FROM accounts WHERE id = 1 FOR UPDATE NOWAIT;
```

**SKIP LOCKED**
`SELECT * FROM <表名> WHERE <条件> FOR UPDATE SKIP LOCKED`
```sql
-- 跳过已锁行
SELECT * FROM accounts WHERE id = 1 FOR UPDATE SKIP LOCKED;
```

## Advisory 锁

**会话级 Advisory 锁**
`pg_advisory_lock(<key>)` / `pg_try_advisory_lock(<key>)` / `pg_advisory_unlock(<key>)`
```sql
-- 获取锁（阻塞等待）
SELECT pg_advisory_lock(12345);

-- 尝试获取（非阻塞）
SELECT pg_try_advisory_lock(12345);

-- 释放锁
SELECT pg_advisory_unlock(12345);
```

**事务级 Advisory 锁**
`pg_advisory_xact_lock(<key>)`
```sql
-- 事务结束自动释放
SELECT pg_advisory_xact_lock(12345);
```

**双 int4 参数**
`pg_advisory_lock(<classId>, <objId>)`
```sql
-- 使用双参数
SELECT pg_advisory_lock(1, 100);
```

**应用场景**
`SELECT CASE WHEN pg_try_advisory_lock(<key>) THEN 'acquired' ELSE 'locked' END`
```sql
-- 分布式锁、限流
SELECT CASE WHEN pg_try_advisory_lock(12345)
  THEN 'acquired'
  ELSE 'locked'
END;
```

## 死锁处理

**按固定顺序访问资源**
`UPDATE <表名> SET ... WHERE id = <小ID>; UPDATE <表名> SET ... WHERE id = <大ID>`
```sql
-- 始终按 id 升序更新
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
```

**使用显式锁预先锁定**
`SELECT * FROM <表名> WHERE id IN (...) ORDER BY id FOR UPDATE`
```sql
-- 预先锁定
BEGIN;
SELECT * FROM accounts WHERE id IN (1, 2) ORDER BY id FOR UPDATE;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
```

**设置锁等待超时**
`SET lock_timeout = '<时间>'`
```sql
-- 设置锁等待超时
SET lock_timeout = '5s';
```

## VACUUM 机制

**普通 VACUUM**
`VACUUM <表名>`
```sql
-- 标记死元组空间为可重用
VACUUM accounts;
```

**VACUUM ANALYZE**
`VACUUM ANALYZE <表名>`
```sql
-- 同时更新统计信息
VACUUM ANALYZE accounts;
```

**VACUUM FULL**
`VACUUM FULL <表名>`
```sql
-- 回收磁盘空间（需排他锁，阻塞所有操作）
VACUUM FULL accounts;
```

**并行 VACUUM**
`VACUUM (PARALLEL <进程数>) <表名>`
```sql
-- 并行 VACUUM（PostgreSQL 13+）
VACUUM (PARALLEL 4) accounts;
```

**查看死元组统计**
`SELECT relname, n_live_tup, n_dead_tup, last_vacuum, last_autovacuum FROM pg_stat_user_tables ORDER BY n_dead_tup DESC`
```sql
-- 查看表的死元组统计
SELECT relname, n_live_tup, n_dead_tup,
  last_vacuum, last_autovacuum, last_analyze, last_autoanalyze
FROM pg_stat_user_tables
ORDER BY n_dead_tup DESC;
```

## 事务 ID 冻结

**手动冻结**
`VACUUM FREEZE <表名>`
```sql
-- 手动冻结
VACUUM FREEZE accounts;
```

**查看冻结状态**
`SELECT relname, age(relfrozenxid) as xid_age, pg_size_pretty(pg_total_relation_size(oid)) as size FROM pg_class WHERE relkind = 'r' ORDER BY age(relfrozenxid) DESC`
```sql
-- 查看表的冻结年龄
SELECT relname, age(relfrozenxid) as xid_age,
  pg_size_pretty(pg_total_relation_size(oid)) as size
FROM pg_class
WHERE relkind = 'r'
ORDER BY age(relfrozenxid) DESC;
```

**查看数据库冻结年龄**
`SELECT datname, age(datfrozenxid) as xid_age FROM pg_database ORDER BY age(datfrozenxid) DESC`
```sql
-- 查看数据库级别的冻结年龄
SELECT datname, age(datfrozenxid) as xid_age
FROM pg_database
ORDER BY age(datfrozenxid) DESC;
```

## 表级自动清理配置

**自定义自动清理参数**
`ALTER TABLE <表名> SET (<参数> = <值>[, ...])`
```sql
-- 表级自定义自动清理
ALTER TABLE orders SET (
  autovacuum_vacuum_scale_factor = 0.05,
  autovacuum_analyze_scale_factor = 0.02,
  autovacuum_vacuum_cost_delay = 1ms
);
```

**禁用自动清理**
`ALTER TABLE <表名> SET (autovacuum_enabled = false)`
```sql
-- 禁用某表自动清理（不推荐）
ALTER TABLE archive_table SET (autovacuum_enabled = false);
```
